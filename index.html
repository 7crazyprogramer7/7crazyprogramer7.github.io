在springmvc.xml下配置

<bean id="mappering" class="org.springframework.web.servlet.handler.SimpleUrlHandlerMapping"> <property name="urlMap"> <map> <entry key="demo" value-ref="democon"></entry> </map> </property> </bean>

3 HandlerAdepter:调用方法

<bean id="adapter" class="org.springframework.web.servlet.mvc.SimpleControllerHandlerAdapter"></bean>

  • 可不配置，使用默认的
  • 自定义

          <property name="prefix" value="/WEB-INF/"></property>

  </bean>`
* 此处prefix的值为/WEB-INF/是因为jsp文件在WEB-INF下

Android standard

发表于 2019-03-07

title: Android 的四种launchMode（启动模式）：

standard ：

既使当a活动处于栈顶时，仍会创建一个新的a活动。

singleTop：

当a活动处于栈顶时，这时再启动a活动时不会再创建新的，但a不在栈顶时会再创建一个新的a活动。

singleTask：

俩个活动在一个任务栈中，先启动a活动（设a活动为main活动），（a活动进栈），在启动b活动（b活动进栈），b在a上，此时在启动a活动，b活动会被弹出，任务栈中只有a活动一个，若此时按back键会直接退出程序（app）

singleInstance：

两个活动在不同的task stack 中，且每个栈中只包括一个活动

spring 的自动注入(针对ref的注入形式)

1. autowire标签（在spring容器中找）、
  • default表示不自动注入
  • buName表示找与要注入的对象名相同的bean的id
  • byType表示找与要注入的对象相同的bean的class类型（注意在使用时bean应唯一）
  • constructor表示用构造方法注入底层用byName（构造方法的参数对象名与bean的id相同）
  • 在beans中加default-autowire标签配置全局的注入方法
    2. 一般不删除sqlSessionFatory的配置信息的原因

    注入

    3. spring 自动注入的一个巨坑
    • 当未添加属性文件扫描，并开启byNmae的自动注入机制时
    • 配置 sqlsessionfactory的bean的id值可以是sqlsessionfactory，这时会根据byName自动注入依赖
    • 当 添加属性文件扫描，并开启byNmae的自动注入机制时
      • default-autowire="byName"
      • 配置 sqlsessionfactory的bean的id值不可以是sqlsessionfactory ，
      • 且<property name="sqlSessionFactory" ref="factory"></property>
        这行代码要改为
        <property name="sqlSessionFactoryBeanName" value="factory"></property>
    • 原因及其解释
      • 由于 default-autowire="byName"的级别十分高，高于属性文件
        的加载级别

在扫描时，当扫描器到<bean class="org.mybatis.spring.mapper.MapperScanner Configurer">
时，若sqlsessionfactory的bean的id值是sqlsessionfactory，由于自动注入会一层层向上实例化，到达

时由于属性文件未加载而报错
当然若不有byName自动注入，用<property name="sqlSessionFactory" ref="factory"></property>ref的注入形式同样有以上问题，即属性文件未加载
改用<property name="sqlSessionFactoryBeanName" value="factory"></property>加载时实例化sqlsessionfactory时会等待dataresource实例化完成，则不报错。

spring的:Caused by: org.springframework.beans.factory.BeanNotOfRequiredTypeException: Bean named ‘‘ is expected to be of type ‘‘ but was actually of type ‘com.sun.proxy.$Proxy**’的问题原因及解决方法

1. 可能的原因：pointcut-ref写成了pointcut
2. 解决 ：在applicationContext.xml文件里面的

<!--开启基于注解的事务，使用xml配置形式的事务（必要主要的都是使用配置式） --> <aop:config> <!-- 切入点表达式 --> <aop:pointcut expression="execution(* com.qihang.service..*(..))" id="txPoint"/> <!-- 配置事务增强 --> <aop:advisor advice-ref="txAdvice" pointcut-ref="txPoint"/> </aop:config>

后加上

<aop:aspectj-autoproxy proxy-target-class="true"/>

mybatis与spring整合因别名使用导致的classnotfound或org.springframework.beans.factory.BeanCreationException问题解决

发表于 2019-02-28

mybatis与spring整合是别名使用导致的classnotfound或org.springframework.beans.factory.BeanCreationException问题解决

1. 命名空间没写对或不全，如果使用mapper，则

<mapper namespace="com.sitech.mapper.StudentMapper">

命名空间要全
2. 在mapper.xml文件中的resultType标签值要完整包名

<select id="checkmessage" resultType="cn.cq.pojo.Student">

3. 在配置sqlsessionfactory的bean中交一个property标签

<property name="typeAliasesPackage" value="cn.cq.pojo"></property>

这样就为pojo下所有的类配置了别名，即为类名

使用Hexo

1. hexo的简单使用

  • hexo是什么？

    heox is A fast, simple & powerful blog framework ，我们通过hexo搭建的博客然后部署到 github 上 ，别人通过 github 主页可以访问到你搭建的博客。

  • 前期准备

    • 首先安装一个 git 和 注册一个 github 账号(可点击此处自行安装)
    • 然后安装 Node.js 。（请自行安装，过程很简单）

  • 安装 Hexo （注意：以下所要输入的皆为命令行）

    1. 在D盘中（建议在非系统盘）建一个文件夹
      • 在文件夹中点击右键，选择git bash here.
      • 然后输入 npm install -g hexo-cli 即可。
    2. 再输入hexo init 进行初始化，然后输入hexo install 安装其他一些可用的文件包。
    3. 安装 deployer-git 插件：继续输入npm install hexo-deployer-git –save

  • 建立GitHub项目

    为项目命名，格式为（用户名 + github.io）

  • 修改_config.yml 文件

    • 在GitHub上复制生成的地址
    • 打开_config.yml文件,改deploy下的type后为git，repo后为复制的地址，branch后为manster
  • 再输入hexo g -d, 然后在网页上输入https://你的github用户名.ginhub.io即可。

发表于 2019-02-28

mybatis与spring整合是别名使用导致的classnotfound或org.springframework.beans.factory.BeanCreationException问题解决

1. 命名空间没写对或不全，如果使用mapper，则

<mapper namespace="com.sitech.mapper.StudentMapper">

命名空间要全
2. 在mapper.xml文件中的resultType标签值要完整包名

<select id="checkmessage" resultType="cn.cq.pojo.Student">

3. 在配置sqlsessionfactory的bean中交一个property标签

<property name="typeAliasesPackage" value="cn.cq.pojo"></property>

这样就为pojo下所有的类配置了别名，即为类名

<mapper namespace="com.sitech.mapper.StudentMapper">

2. 原因是mybatis.cfg.xml配置文件缺少了对Mapper.xml的引用加上

<mapper resource="cn/cq/mapper/SchoolCardMapper.xml"/>

3. mapper.xml文件名和所写的接口名称不相同
4. 接口中没有此方法
5. 若使用

<package name="cn.cq.mapper"/>
一定在mapper包下加

此图中的shoolcardmapper.java文件

6. 或者直接用

使用Hexo

1. hexo的简单使用

  • hexo是什么？

    heox is A fast, simple & powerful blog framework ，我们通过hexo搭建的博客然后部署到 github 上 ，别人通过 github 主页可以访问到你搭建的博客。

  • 前期准备

    • 首先安装一个 git 和 注册一个 github 账号(可点击此处自行安装)
    • 然后安装 Node.js 。（请自行安装，过程很简单）

  • 安装 Hexo （注意：以下所要输入的皆为命令行）

    1. 在D盘中（建议在非系统盘）建一个文件夹
      • 在文件夹中点击右键，选择git bash here.
      • 然后输入 npm install -g hexo-cli 即可。
    2. 再输入hexo init 进行初始化，然后输入hexo install 安装其他一些可用的文件包。
    3. 安装 deployer-git 插件：继续输入npm install hexo-deployer-git –save

  • 建立GitHub项目

    为项目命名，格式为（用户名 + github.io）

  • 修改_config.yml 文件

    • 在GitHub上复制生成的地址
    • 打开_config.yml文件,改deploy下的type后为git，repo后为复制的地址，branch后为manster
  • 再输入hexo g -d, 然后在网页上输入https://你的github用户名.ginhub.io即可。
